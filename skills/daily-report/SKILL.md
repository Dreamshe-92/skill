# Daily Report Skill

自动生成周报总结的技能，融合 SRE 和运维工程的专业视角。

## 功能说明

此技能从当前目录下的 `daily/` 目录中自动读取本周的打卡记录文件，按项目和日期整理工作内容，生成结构化的周报总结。特别针对 SRE 和运维工程师的工作特点，提供专业化的汇报框架。

## 文件格式

打卡记录文件命名格式：`yyyymmdd`（例如：20260108）

### 基础格式示例
```

1、工作内容描述
2、工作内容描述

WEAPM:
1、工作内容描述

WEPS:
1、工作内容描述
```

## 使用流程

### 步骤 1：获取本周打卡文件

使用脚本自动获取本周的所有打卡记录：

```bash
python scripts/get_weekly_files.py
```

该脚本会：
- 扫描当前目录下的 `daily/` 目录
- 识别本周一到本周日的所有文件
- 按日期顺序显示文件内容

### 步骤 2：读取并整理内容

读取每个文件的内容，提取以下信息：
- 日期（从文件名获取）
- 项目分类（WCS、WEAPM、WEPS 等）
- 具体工作内容
- SRE 指标数据（如有）
- 运维专项工作（如有）

### 步骤 3：生成周报

根据工作内容类型选择合适的周报结构：

## SRE/运维 专业指导原则

### 🎯 工作原则（SRE 视角）

1. **数据驱动**: 用 SLI/SLO、MTTR、MTBF 等量化指标说明工作价值
2. **预防为主**: 强调巡检、监控、容量规划等预防性工作
3. **自动化优先**: 突出自动化工具开发对效率的提升
4. **持续改进**: 展示从故障中学习并优化系统的过程
5. **业务价值**: 将技术工作与业务可用性、用户体验关联
6. **透明沟通**: 如实报告问题，不隐瞒风险

### 📊 关键指标说明

**稳定性指标**
- **可用性** (Availability): 系统正常运行时间占比
- **MTTR** (Mean Time To Recover): 平均故障恢复时间
- **MTBF** (Mean Time Between Failures): 平均故障间隔时间
- **SLI** (Service Level Indicator): 服务水平指标
- **SLO** (Service Level Objective): 服务水平目标
- **SLA** (Service Level Agreement): 服务水平协议

**性能指标**
- **延迟** (Latency): P50、P95、P99 延迟
- **吞吐量** (Throughput): QPS/TPS
- **错误率** (Error Rate): HTTP 4xx/5xx 比例

**变更指标**
- **变更频率**: 每周/每月变更次数
- **变更成功率**: 成功变更占总变更比例
- **回滚率**: 需要回滚的变更比例

**效率指标**
- **自动化率**: 自动化任务占总任务比例
- **人工干预率**: 需要人工处理的告警比例
- **Toil** (运维琐事): 重复性手动工作时间占比

### 📝 报告撰写技巧

**突出量化成果**
```
❌ 差：优化了监控系统
✅ 好：优化告警规则，误报率从 50% 降至 20%，每周减少无效告警 150+ 条
```

**关联业务价值**
```
❌ 差：升级了 Kubernetes 版本
✅ 好：升级 K8s 至 v1.26，支持新特性使部署速度提升 40%，缩短发布窗口 30 分钟
```

**展示系统性思维**
```
❌ 差：修复了 Redis 慢查询
✅ 好：
- 根因分析：慢查询由大 key 导致
- 临时措施：清理大 key，性能恢复
- 长期方案：开发大 key 监控告警，制定数据规范
- 效果验证：P99 延迟从 2s 降至 50ms
```

- **总结反思**: 亮点、改进空间、风险问题

## 注意事项

### 文件格式
- 确保打卡文件格式正确（yyyymmdd.txt）
- 文件内容使用 UTF-8 编码
- 项目分类使用大写字母加冒号格式（如 "WCS:"）
- 每个工作项使用编号（1、2、3...）

### 内容质量
- **真实性**: 数据和指标必须真实可验证
- **时效性**: 周报应在周五下班前或周一上午提交
- **完整性**: 不遗漏重要工作，特别是故障和变更
- **简洁性**: 突出重点，避免流水账

### SRE/运维特有注意事项
- **指标口径**: 统一指标计算口径（如 MTTR 从发现到恢复还是从响应到恢复）
- **敏感信息**: 避免泄露生产环境具体IP、密码等敏感信息
- **客观性**: 如实报告问题，不过度修饰
- **可追溯**: 重大工作应有关联工单、文档链接
